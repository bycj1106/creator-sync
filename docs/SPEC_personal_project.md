### 任务3：数据同步方案 ⏳待开发

**需求背景：**
本地用户（不注册的用户）的数据存储在本地，而注册用户的数据存储在云端。需要设计一个合理的同步方案。

**同步方案设计：**

| 场景 | 方案描述 |
|------|----------|
| 本地→云端 | 用户绑定云端账号时，可选择将本地数据合并到云端 |
| 云端→本地 | 用户登录时自动拉取云端数据到本地（需用户确认） |
| 冲突处理 | 以云端数据为准，本地可选择保留或覆盖 |
| 离线支持 | 本地用户离线时可正常使用，联网后同步 |

**数据模型：**

```javascript
// 用户类型标识
UserType: 'local' | 'cloud'

// 本地用户数据结构
{
  id: string,
  type: 'local',
  username: string,
  createdAt: timestamp,
  data: {
    plans: [...],
    tasks: [...],
    inspirations: [...]
  }
}

// 云端用户数据结构
{
  id: string,
  type: 'cloud',
  username: string,
  cloudId: string,  // 云端用户ID
  createdAt: timestamp,
  lastSyncAt: timestamp
}
```

**同步流程：**

1. **本地用户注册云端账号时：**
   - 用户选择"绑定云端账号"
   - 输入云端用户名密码验证
   - 提示"是否合并本地数据"
   - 合并后本地数据上传到云端

2. **云端用户本地登录时：**
   - 自动拉取最新云端数据
   - 显示"数据同步中"提示
   - 同步完成后显示最新数据

**验收条件：**
1. 本地用户可以绑定云端账号
2. 绑定时可选择是否合并本地数据
3. 云端用户登录时自动同步数据
4. 冲突时以云端数据为准
5. 同步状态有明确提示
