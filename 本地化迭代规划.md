# 创作者同步助手 - 本地化迭代规划（简化版）

## 一、需求概述

简化本地化方案：

1. 登录页添加"本地一键登录"按钮
2. 本地用户名为"本地用户"
3. 数据存储在浏览器本地（LocalStorage/IndexedDB）
4. 邀请码仅用于开启云同步服务，**仅管理员可生成**
5. 云同步仅提供给**朋友**使用，非盈利目的，控制服务器成本

---

## 二、架构设计

### 2.1 用户类型

| 类型 | 用户名 | 数据存储 | 云同步 |
|------|--------|---------|--------|
| 本地用户 | 本地用户 | 浏览器本地 | ❌ |
| 云同步用户 | 自定义 | 本地 + 后端 | ✅ |

### 2.2 数据流

```
本地用户模式：
  数据 → LocalStorage → 无需后端

云同步用户模式：
  数据 → IndexedDB ←→ 后端API
```

### 3.3 本地数据存储

使用 LocalStorage 存储核心数据：

```typescript
// 存储数据
function saveLocalData(key: string, data: any) {
  localStorage.setItem(key, JSON.stringify(data));
}

// 读取数据
function getLocalData(key: string) {
  const data = localStorage.getItem(key);
  return data ? JSON.parse(data) : null;
}

// 存储结构
const STORAGE_KEYS = {
  USER: 'creator-sync-user',
  PLANS: 'creator-sync-plans',
  TASKS: 'creator-sync-tasks',
  INSPIRATIONS: 'creator-sync-inspirations',
  SETTINGS: 'creator-sync-settings'
};
```

### 3.5 邀请码机制（管理员生成，仅供朋友使用）

#### 3.5.1 设计目标
- 非盈利目的
- 仅提供给朋友使用
- 控制服务器成本

#### 3.5.2 邀请码由管理员生成
```
管理员操作（后端）：
1. 访问管理员专属接口
2. 生成邀请码（如：CS-ABC123）
3. 将邀请码告知朋友
4. 朋友使用邀请码开启云同步
```

#### 3.5.3 邀请码管理接口（后端）

```javascript
// POST /api/admin/generate-invite
// 生成邀请码（需管理员密码）
{
  "adminPassword": "xxxxx",      // 管理员密码
  "maxUses": 1,                  // 使用次数，默认1
  "expiresIn": null               // 过期时间，可选
}

// GET /api/admin/invite-codes
// 查看所有邀请码

// DELETE /api/admin/invite-codes/:code
// 删除邀请码
```

#### 3.5.4 邀请码存储

```sql
-- 后端数据库
CREATE TABLE invite_codes (
  code TEXT PRIMARY KEY,
  max_uses INTEGER DEFAULT 1,
  used_count INTEGER DEFAULT 0,
  created_by INTEGER,
  created_at TEXT,
  expires_at TEXT
);
```

#### 3.5.5 使用流程

```
开启云同步流程：
1. 用户点击"开启云同步"
2. 提示"需要邀请码，是否继续？"
3. 用户输入邀请码
4. 前端验证邀请码格式（CS-开头）
5. 后端验证邀请码有效性
6. 验证通过 → 跳转到注册/登录
7. 完成后本地数据与云端同步
```
